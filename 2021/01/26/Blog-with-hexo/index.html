hexo로 블로그 만들기 과정 정리

수업시간에 윈도우에서 git과 hexo를 설치하여.. 2시간 넘는 시행착오를 겪으며
Ubuntu환경에서 새로 해보면서 정리해보려고 한다.

1. 준비환경 만들기
  • Node.js 설치
    1. curl 설치
      Node.js를 url을 통해 다운 받기 위해 필요함
    2. curl 설치 확인
    3. PPA를 이용하여 Node.js 14.x 버전 설치
    4. npm을 위한 build-essential 패키지 설치
    5. Node.js와 npm이 잘 설치되었는지 확인
  • vim 설치
    vi의 호환 텍스트용 편집기로서 자유롭게 편집환경을 변경할 수 있음
  • hexo 설치
    1. npm명령어를 사용하여 설치(npm 을 통해 hexo-cli 모듈을 global하게 설치한다는 뜻)
    2. 블로그로 사용할 폴더 생성과 접근 후, 초기화에 필요한 패키지 설치
    3. 로컬서버 띄워보기(ctrl + c로 끄기)
  • git 계정, 빈 repository 생성, set configuration
    해당 내용은 아래 포스트 확인(추후 업로드 예정)
2. hexo설정(github에 배포하기 전에 연동하기)
  • _config.yml을 통해 hexo와 github 연동
    hexo를 사용하면 별도의 git명령어(add, commit, push) 없이 알아서 github에 업로드를 해줌. hexo와 github 간의 연결고리가 있어야 하는데 _config.yml을 통해 지정
  • hexo에서 github로 배포할 수 있는 플러그인 설치
3. 새 글 작성

아래 명령어를 실행하면 ~/source/_post/ 아래에 글제목.md가 생성되고, vim으로 내용을 작성하면 됨. 마크다운 문법은 검색 추천!

4. 빌드, 로컬에서 확인, 배포, 실제 블로그에서 확인
  • 마크다운으로 작성된 파일을 html파일로 바꿔주기 위해
  • 글이 잘 작성되었는지 로컬서버를 띄워 확인
  • 끝으로 배포! Username과 Password를 입력하면 github에 업로드
  • http://계정명.github.io로 접속하여 확인(업로드까지 2~3분 소요될 수 있음)
