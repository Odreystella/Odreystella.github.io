    문제 설명

    배열 array의 i번째 숫자부터 j번째 숫자까지 자르고 정렬했을 때, k번째에 있는 수를 구하려 합니다.

    예를 들어 array가 [1, 5, 2, 6, 3, 7, 4], i = 2, j = 5, k = 3이라면

    1. array의 2번째부터 5번째까지 자르면 [5, 2, 6, 3]입니다.
    2. 1에서 나온 배열을 정렬하면 [2, 3, 5, 6]입니다.
    3. 2에서 나온 배열의 3번째 숫자는 5입니다.
      배열 array, [i, j, k]를 원소로 가진 2차원 배열 commands가 매개변수로 주어질 때, commands의 모든 원소에 대해 앞서 설명한 연산을 적용했을 때 나온 결과를 배열에 담아 return 하도록 solution 함수를 작성해주세요.
    제한사항
    • array의 길이는 1 이상 100 이하입니다.
    • array의 각 원소는 1 이상 100 이하입니다.
    • commands의 길이는 1 이상 50 이하입니다.
    • commands의 각 원소는 길이가 3입니다.
    입출력 예
    array commands return
    [1, 5, 2, 6, 3, 7, 4] [[2, 5, 3], [4, 4, 1], [5, 6, 3]] [5, 6, 3]
    입출력 예 설명

    [1, 5, 2, 6, 3, 7, 4]를 2번째부터 5번째까지 자른 후 정렬합니다. [2, 3, 5, 6]의 세 번째 숫자는 5입니다.
    [1, 5, 2, 6, 3, 7, 4]를 4번째부터 4번째까지 자른 후 정렬합니다. [6]의 첫 번째 숫자는 6입니다.
    [1, 5, 2, 6, 3, 7, 4]를 1번째부터 7번째까지 자릅니다. [1, 2, 3, 4, 5, 6, 7]의 세 번째 숫자는 3입니다.

    배운 점

    • for문을 쓸 일이 있을 때는 map을 떠올리자.
    • 어떤 작업을 반복해야 한다면 lambda로 만들어보자.
    • list(map(lambda 함수, 이터러블 중 시퀀스 객체(e.g., list, str, tuple, range)) 구조 기억하기.
    • sort()는 원본 배열 자체를 정렬하고, None을 리턴한다.
      즉, new_array = array.sort()하면 new_array에는 None가 담긴다.
    • sorted()는 원본을 훼손하지 않고, 정렬한 새로운 배열을 리턴한다.
    • list의 경우, 속도는 **sort()**가 sorted()보다 빠르다고 한다.
      sorted()는 복사본을 만들기 때문, 미세한 차이지만 sort()쓴 ver.1코드에서 0.01ms걸리는 케이스가 더 적게 나온다..ㅋ
    • sort()는 list 자료형만 쓸 수 있고, sorted()는 tuple, dict, str, set 등 이터러블 객체라면 모두 사용 가능하다.
    • 이터러블 객체 : list, tuple, range, str, dict, set
    • 시퀀스 객체 : list, tuple, range, str
