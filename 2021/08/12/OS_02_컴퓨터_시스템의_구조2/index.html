직접 I/O장치에 접근이 불가능한 사용자 프로그램은 어떻게 I/O 작업을 하나?
  • I/O를 하기 위해서 OS에 해당하는 메모리 주소로 넘어가야 하는데 mode bit이 1이기 때문에 사용자 프로그램이 직접 접근하는 것은 허용되지 않음
  • 사용자 프로그램이 I/O 작업이 있을 때 트랩을 이용해서 직접 인터럽트를 발생시킴(OS한테 시스템 콜)
  • 인터럽트가 발생하면 mode bit이 1에서 0으로 바뀌고 CPU 제어권이 프로그램에서 OS로 넘어감 → 인터럽트 벡터가 가르키는 인터럽트 서비스 루틴으로 이동함
  • OS는 올바른 I/O 요청인지 확인 후 올바르면 device controller에게 I/O 요청 → I/O 요청이 끝나면 device controller이 인터럽트를 검(하드웨어 인터럽트)
