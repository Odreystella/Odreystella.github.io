운영체제란?
  • 컴퓨터 하드웨어 바로 위에 설치되어 사용자 및 다른 모든 소프트웨어와 하드웨어를 연결하는 소프트웨어 계층
  • 사용자는 하드웨어를 직접 사용하지 않고 그 위에 있는 운영체제와 소통하게 됨
  • 운영체제의 목표 :
    1. 사용자에게 컴퓨터 시스템을 편리하게 사용할 수 있는 환경 제공
    2. 컴퓨터 시스템의 자원(CPU, 메모리, 입출력 장치 등)을 효율적으로 관리
커널?
  • 좁은 의미의 운영체제를 말함. 운영체제의 핵심 부분으로 메모리에 상주함
운영체제 분류
  • 동시 작업 가능 여부 :
    1. 단일 작업 : 한 번에 하나의 작업만 처리
    2. 다중 작업 : 동시에 두 개 이상의 작업 처리
  • 사용자의 수 :
    1. 단일 사용자
    2. 다중 사용자 : 원격 서버에 여러명의 사용자가 접근이 가능함
  • 처리 방식 :
    1. 일괄 처리(배치 시스템) : 작업 요청의 일정량을 모아서 한번에 처리, 처리 완료 될때까지 다른 작업 진행되지 않음 ex)omr 카드 처리기, punch card 처리 시스템
    2. 시분할 : 컴퓨터 처리 능력을 일정한 시간 단위로 분할하여 사용, 짧은 응답시간을 가지면서 주어진 자원을 최대한 활용하는 것이 목적, 1초 내 응답 등 정확한 시간내에 응답을 하는 목적은 아니라 사용자가 많으면 느릴 수 있음, 범용 컴퓨터에서 사용, 영화나 네비게이션등의 응용프로그램은 실시간으로 정보를 제공해야 하지만 시분할 os 위에 돌아가기 때문에 보완이 필요함
    3. 실시간 : 정해진 시간 안에 어떠한 일이 반드시 종료되는걸 목적으로 함, 특수한 목적을 가진 컴퓨터에서 사용 ex) 반도체 장비, 미사일 제어 등
멀티태스킹? 멀티프로그래밍? 멀티 프로세싱? 용어 정리
  • 멀티태스킹 : 짧은 시간으로 분할해서 여러 작업이 동시에 진행되는 것처럼 보임, 실제로 CPU는 매 순간 하나의 작업만 실행함. 하나의 프로그램이 끝나기 전에 다른 프로그램 실행 가능
  • 멀티프로그래밍 : 메모리에 여러 프로그램이 동시에 올라가는 것을 강조(메모리 측면 강조), 멀티태스킹이 되려면 멀티프로그래밍이 되어야 함
  • 시분할 시스템 : CPU의 시간을 분할하여 나누어 쓴다는 의미를 강조(CPU 측면 강조)
  • 멀티프로세스 : 실행중인 프로그램을 의미하는 프로세스가 여러개라는 의미로 여러 프로그램이 동시에 실행되는 것을 말함
  • — 위의 용어들은 CPU가 하나 있는 환경임
  • cf)멀티프로세서(다중처리기) : 하나의 컴퓨터에 여러개의 CPU가 있는 컴퓨터, 하드웨어적으로 다른 시스템
운영체제 예시
  • 유닉스 :
    • 대형 컴퓨터, 서버를 위한 OS
    • 처음부터 멀티 태스킹이 가능했음
    • 어셈블리어로 개발하기 어려워 C언어를 만들어서 OS 개발
    • 소스 코드 공개
    • c언어로 개발되어 다른 기계에서도 동작 가능함
    • 최소한의 커널구조로 메모리 사용을 최소화함
    • ex) System V, SunOS, Linux (독특하게 개인용 컴퓨터도 사용 가능) 등
  • MS DOS → Ms Windows
    • 개인용 컴퓨터를 위한 OS
    • 처음에는 단일 사용자, 단일 작업만 가능했음
    • 소스 코드 비공개, 만들어진 결과물만 유료로 제공
    • 메모리 크기가 640KB.. 매우 작음
