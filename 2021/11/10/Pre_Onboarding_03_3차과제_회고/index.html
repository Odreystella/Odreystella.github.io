3차 과제의 요구사항을 간략하게 정리해보자

과제의 기한과 요구사항은 다음과 같았다.

  • 기한: 21/11/8 16:00 ~ 11/10 10:00 (약 1.5일)
  • 개발 요구사항:
    • 선호 기술 스택: flask or fastapi
    • DB: RDBMS 사용
    • REST API 기능
      • 회사명 자동 완성
        • 회사명 일부만 들어가도 검색이 되어야 함
      • 회사 이름으로 회사 검색
      • 새로운 회사 추가
    • 제공되는 test case를 통과할 수 있도록 개발해야 함
    • ORM 사용
    • 결과는 JSON 형태
    • Docker 사용 시 가산점 있음
어떻게 과제를 해나갔는가

2번의 과제를 진행하면서 우리는 주어진 시간이 매우 빠르게 지나감을 절실히 깨달았고, 또다시 밤을 새지 않기 위해 기능과 업무를 분담해서 진행하기로 했다. 그리고 어느 정도 일정 분배에 노하우가 생겼다. 첫날엔 git projects와 issues 작성, 프로젝트 초기환경 셋팅, 모델링 토론 및 완성하여 가능한 migration까지 진행, 둘째날엔 기능 개발 및 테스트코드 작성, 배포, API문서 작성, README.md 작성 이렇게 말이다.

이번 과제는 3개의 기능 개발이 요구되었고, 나는 다른 팀원과 새로운 회사 추가 기능을 맡게 되었다. 우리는 둘째날 점심부터 구글 미트로 만나 약 6시간동안 코드를 작성하고 테스트했다.

어려웠던 점은?

이번 과제는 어려움이 많았는데 가장 어려웠던 점은 모델링이었다. 주어진 데이터 셋을 보고 테이블을 설계하고 정규화하다 보니 회사, 언어코드, 태그 3가지로 테이블을 나눴는데 이들의 관계를 모두 N:M으로 설계했다. 이렇게 설계하는게 맞나 의심을 하면서 최종적으로 결정하기까지 시간이 오래 걸렸기 때문에 일단 진행하기로 했는데, 기능을 구현하면서도 테이블 구조 때문에 복잡함을 많이 느꼈다. 이 부분은 리뷰를 받고 모델링을 개선해서 다시 구현해보고 싶다.

두번째는 새로운 프레임워크인 flask를 사용하다 보니 모델 작성, url 설정, 뷰 코드 작성, 로컬 서버 실행 등 모든 부분이 새로웠다. 또한, Django보다 구조가 간단하다 보니 간편하면서도 좋은 앱을 만들기 위해 우리가 구조를 잡아야 했고, SQLAlchemy ORM을 적용하는 부분도 공식 문서를 잘 이해해야 했다.

마지막으로 Docker로 배포하는 부분이 어려웠다. 2차 과제때, Django를 Docker로 배포해봐서 금방 할 것이라 생각했는데 Docker, flask, Gunicorn(WSGI), Nginx의 전체 흐름을 파악하지 못하다 보니 많은 시도에도 불구하고 결과적으로는 성공하지 못했다. 이 부분도 좀 더 공부한 후에 꼭 성공시켜보고 싶다.

무엇을 느끼고 무엇을 배웠나

이번 과제도 Docker로 배포를 시도할 때 시간을 많이 쓰다보니 제출 당일 새벽 6시에 과제를 완료했다. 2차과제 때보다 1시간.. 단축했다는 점에서 자축했지만… 또 밤을 새서 슬펐다 ㅠㅠ 역시 계획은 계획대로 안되어야 제맛인가,,, 어려움이 많았던 과제였던 만큼 앞으로 공부할 부분이 많다는 것을 느꼈지만 새로운 프레임워크, 새로운 ORM을 사용해봐서 재밌었다. 그리고 git rebase, PR 등 git으로 팀원들과 협업하는 부분도 처음보다는 나아지고 있는 것 같다.

4차 과제는 어떻게 하고 싶은가

여전히 가장 큰 목표는 시간을 단축하여 제출 당일 새벽 1시 전에는 자는 것이고, 기본 요구사항을 일찍 끝내고 가산점을 주는 부분 또한 시도해서 과제의 완성도를 높이고 싶다!!
