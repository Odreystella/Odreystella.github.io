git fork와 git clone의 차이점

  1. 저장 장소

    • Forking을 하면 Github 계정에 원본 레포지토리(upstream repository)의 복사본을 생성한다.
    • Cloning을 하면 내 로컬 컴퓨터에 원본 레포지토리의 복사본을 생성한다. Forking을 하고 Cloning하거나 Github에 새로운 레포지토리를 만들고 Cloning를 하는 방법이 있다.

  2. 원본 저장소와의 병합

    • Forking을 하면 변경 사항에 대해 원격 저장소(원본 레포지토리)로 pull request 를 보내, 요청이 받아들여질 경우 병합된다.
    • Cloning하여 생성된 로컬 저장소를 git remote add 하여 내 Github의 원격 저장소와 연결한다면, 변경 사항을 직접 내 저장소에 push 할 수 있다. 원격 저장소가 다르다면 pull request를 보내게 된다.

  3. 명령어 유무

    • Forking은 개념이고 Cloning은 프로세스이다.
    • Forking은 관련된 명령이 없고, Cloning는 git clone 명령을 통해 이루어진다.
Github에서 fork 및 clone를 사용한 프로세스 흐름

  1. Forking 후 Git Repo 복제

    • 1단계: fork하면 내 Github 계정에 레포지토리 복제
    • 2단계: git clone하여 로컬 컴퓨터에 복제(원격저장소는 fork된 저장소와 연결)
    • 3단계: 변경 사항은 내 저장소에 push
    • 4단계: fork한 원본 레포지토리로 pull request

  2. Forking 없이 Git Repo 복제

    • 1단계: 내 Github 계정에 새 레포지토리 생성
    • 2단계: git clone하여 로컬 컴퓨터에 복제(git remote add하여 원격저장소와 연결)
    • 3단계: 변경 사항은 내 저장소에 push
