인증(Authentication)이란?

인증이란 사용자의 신원을 확인하는 절차이다. 회원가입과 로그인을 통해 사용자가 입력한 정보를 가지고 이 사용자가 누구인지, 우리 서비스의 회원인지를 확인한다.

인증 방법

  1. Session Based Authentication

    • 특징
      • JWT를 사용하기 전에 주로 세션을 이용해 인증함
      • 서버에서 세션 정보를 관리함
      • Django에서 기본적으로 제공하는 세션 인증을 사용해보면 django_session 테이블이 생성되고, 그 안에 session_key, session_data, expire_date와 같은 필드로 세션 정보가 저장됨
    • 과정
      1. 클라이언트에서 로그인 시, 사용자의 인증 정보(id, pw등)를 서버에 전달한다.
      2. 서버는 인증을 처리한뒤 해당 유저에 대해 세션을 생성한다.
      3. 세션 정보는 서버에 저장되고, 클라이언트는 쿠키에 담긴 세션 id를 꺼내 브라우저에 저장한다.
      4. 클라이언트는 이후 어떠한 요청을 보낼 때, 쿠키에 세션 id를 담아 함께 서버에 전달한다.
      5. 서버는 전달받은 세션 id를 이용하여 저장중인 세션 정보로 인증을 처리한다.
      6. 인증에 성공하면 올바른 응답을 보내고, 세션 id가 만료되었을 경우 1번 과정부터 다시 진행된다.
    • 단점
      • 매 요청마다 인증을 위해 데이터베이스를 탐색해야 함(세션정보를 서버에 저장했기 때문)
      • 캐시 등을 이용하여 탐색 과정을 최적화 할 수 있지만 기본적으로 세션 정보가 서버 어딘가에 저장되는 구조임
      • 세션 정보를 저장하는 데이터베이스가 분산되어있을 경우, 각각의 데이터베이스를 탐색해야 하고, MSA(서비스별로 독립적인 구조를 가지는 아키텍쳐)로 설계된 경우 각 어플리케이션 마다 세션 정보를 통합하여 관리하는 것도 번거로움

  2. Token Based Authentication - JWT

    • 특징
      • 서버나, 브라우저가 아닌 토큰 자체에 유저의 정보가 담겨 있음
인가(Authorization)란?

인가는 유저의 요청을 실행할 수 있는 권한이 있는 유저인가를 확인하는 절차이다.
