Python Web Framework Django vs. Flask
  • Django - Full-stack Framework
  • Flask - Micro Framework(Django Rest Framework와 같음)
Django Web Framework
  • Flask보다 약 10배 많은 코드라인으로 개발되어 있다.(code 290, 087 lines)
  • 구글 앱 엔진에서 사용하게 되면서 많은 사람들이 사용하게 되었다.
  • web application을 개발하기 위한 대부분의 기능을 갖추고 있다.
  • ORM(Object relational mapping), 즉 데이터베이스 시스템과 데이터 모델 클래스를 연결하는 기능이 내장되어 있다.
  • 자동으로 관리자 화면을 구성해준다.
  • 게시판 같은 복잡한 구조의 웹을 구성하기에 적합하다.
Falsk Web Framework
  • Micro framework를 기반으로 단순하고 매우 가볍다. (code 28,677 lines)
  • URL 라우팅, Template, Cookie, Debugger 및 개발서버 등 기본 기능만을 제공한다.
  • 최소한의 기능만 제공하기 때문에 데이터베이스 연결, 보안, 인증 등 모두 개발자가 직접 처리해야 한다.
  • 구조는 크게 WSGI용 라이브러리인 Werkzeug와 HTML 렌더링 엔진인 Jinja2 template로 구성되어 있다.
  • 단 10줄도 안되는 코드로 웹 서버를 구동할 수 있지만, 상용 웹 서버를 구현할 때는 단순하지 않다.
  • ORM 기능이 제공되지 않아 SQLAlchemy 등 패키지를 설치하여 사용해야 한다.
  • 단순한 REST API 서버를 만들기에 적합하다.
django와 Flask 차이를 정리하자면?

  1. 프로젝트의 레이아웃

    • django는 한 프로젝트 내에 다양한 어플리케이션이 존재할 수 있다. 그렇기 때문에 같은 프로젝트 내에 여러 어플리케이션이 동시에 사용하는 재사용 가능한 코드와 유지보수가 편하다.
    • Flask는 프로젝트마다 1개의 어플리케이션을 개발하도록 한다.

  2. DB 접근

    • django는 ORM을 제공한다.
    • Flask는 ORM을 제공하지 않기 때문에 SQLAlchemy 등 패키지를 설치해야 한다.

  3. 확장성

    • django는 거의 모든 기능을 지원해주기 때문에 내가 원하는 기능을 따로 추가하고 싶을 때 번거로울 수 있다.
    • Flask는 Micro framework이기 때문에 django보다 유연하고 편하게 확장이 가능하다.

  4. 서비스 목적

    • 여기 4번 항목을 참고한다.

  5. 속도

    • 별반 차이가 없다. 속도가 중요한 서비스라면 다른 언어를 선택하거나 FastAPI를 선택한다.
    • Node.js 아성에 도전한다. FastAPI written in Python
