과제의 선택권

목요일 세션이 끝나고, 2차 과제에 대한 설명을 들었다. 원래는 마피아컴퍼니 과제를 팀원 모두가 참여하고, 프레시코드 과제는 두 팀으로 나누어서 진행하는 것이었는데, 둘 중 하나의 과제만 제출하는 것으로 변경되었다.

우리는 변경사항을 듣기 전에 모르는 기술스택에 대해 미리 공부하기 위해, FastAPI나 Django에서 Neo4J를 어떻게 사용하는지, 그래프 DB는 무엇인지 각자 공부를 하고 있었다(마피아컴퍼니의 과제 요구사항 중 Neo4J DB를 사용하는 것이 필수였다). 구글링 하면서도 그래프DB에 대한 개념이 어려워서 멘붕이었는데, 선택권이 주어져서 다행이었다. 우리는 일단 프레시코드의 과제를 제출하기로 결정했다.

그 다음으로, 우리는 어떤 프레임워크를 사용할지 의견을 나눴다. 그냥 쉽게 우리가 잘 아는 Django를 사용하면 됐었는데, FastAPI에 대해 알게 되니 새로운 프레임워크를 써보는 것도 재밌을 것 같다는 의견이 나왔다. 그렇게 우리는 앞날을 모른 채 신나게투표를 했다.ㅋㅋㅋ flask, fastAPI, DRF, Django 중 DRF가 가장 많은 득표수를 얻고 우리는 DRF에 대해서 공부했다.

2차 과제의 요구사항을 간략하게 정리해보자

과제의 기한과 요구사항은 다음과 같았다.

  • 기한: 21/11/4 17:00 ~ 11/6 10:00 (약 1.5일)
  • 개발 요구사항:
    • DB: RDBMS 사용
    • 로그인 기능
      • JWT 인증 방식 구현
    • 상품 관리 기능
      • 상품 CRUD API 구현
      • 사용자는 상품 조회, admin 권한을 가진 사용자만 상품 추가/수정/삭제 가능
      • 상품 전체목록 조회시, 페이징 처리
    • 적절한 Error Handling
    • 유닛테스트 구현
어떻게 과제를 해나갔는가

우리는 먼저 목요일 밤까지 프레임워크 결정, 업무 분배, 초기 셋팅, 모델링을 하는데 시간을 할애했다.
금요일날엔 12시에 만나 스터디룸을 잡고 models.py를 작성하고 마이그레이션을 했다.그 후, 로그인 기능에 2명, 상품관리 기능에 3명으로 팀을 나눴기 때문에 페어프로그래밍을 진행했다. 나는 jwt 인증 방식을 구현해 보고 싶어서 로그인 기능을 맡아서, DRF에서는 어떻게 구현할 수 있는지 팀원과 레퍼런스를 찾아봤다. DRF에서는 simple jwt를 사용하면 get_token이라는 메서드로 토큰을 알아서 만들어주는 것 같았다. 여러 코드를 참고하며 일단 postman을 돌려봤는데 잘 되지 않았고, 상품 관리 기능 팀도 DRF가 처음이다 보니 serializer를 쓰는 과정에서 에러가 잘 해결되지 않는 상황이었다. 이때 시간은 오후 5시었다.

DRF에 대한 충분한 지식 없이 구글링하면서 구현하다 보니 새로운 내용을 배우는건 좋았는데 기한 안에 과제를 마무리해서 내는 것이 더 중요하다 보니 우리는 다시 Django로 돌아오게 되었다. 각자 맡은 기능을 구현하고 도커로 배포까지 하고 헤어지는것이 목표였는데 error handling, 테스트 코드 짜는 과정에서 시간이 오래 걸리다보니 우리는 다같이 밤을 새게 되었다.

어려웠던 점은?

프리온보딩 전에는 팀 프로젝트 경험이 없다보니 개인적으로 가장 어려운 부분은 협업하는 일이었다. 열심히 하겠다는 의미로 팀장을 자처했는데, 체계적으로 이끌지 못하는 것 같아서 팀원들한테 미안한 마음이 크다. 또한, 팀으로 하기 때문에 맡은 부분에 대해 시간이 너무 길어지면 다른 팀원들도 같이 시간적인 피해를 보게 되는데 2번의 프로젝트를 진행했으니 3차때는 밤을 새지 않기 위해 좀 더 시간 활용을 잘 해야할것 같다.

무엇을 느끼고 무엇을 배웠나

프로젝트 중간에 시간 부족으로 장고 프레임워크로 돌아오긴 했지만, DRF를 쓰기로 했을 때 공식문서를 보면서 어떻게 사용하는지 찾아보고 구현해보는 부분은 좋았다. 잘 되지는 않았지만..ㅎㅎ 프리온보딩에 참여하지 못했다면 장고 프레임워크만 사용할 줄 알았을텐데, 짧은 시간안에 다양한 프레임워크, DB를 사용하라는 과제가 주어지니 공식문서를 봤을 때 두려움이 덜한것 같다.

3차 과제는 어떻게 하고 싶은가

가장 큰 목표는 의사결정을 빨리 하고 밤을 새지 않는 것이다..
