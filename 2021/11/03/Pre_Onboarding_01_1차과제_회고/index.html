11월 1일 월요일, 첫 과제를 받고..

프리온보딩의 첫 번째 과제는 선발과제였던 게시판CRUD API에 추가 기능을 구현하는 것이었다. 게시글 카테고리, 게시글 검색, 대댓글(+pagination), 게시글 읽힘수를 구현하는데, 팀을 반으로 나눠 두개의 결과물을 제출하는 것이 요구사항이었다. 우리 팀은 총 5명이었는데 첫 세션이 끝나고 온라인에서 만나 사다리타기로 팀을 나누었고, 나는 총 2명인 팀에 뽑혔다. 제출기한이 수요일 오전 10시까지였기 때문에 우리는 간단하게 서로의 소개를 마치고, 팀장을 뽑은 후 과제에 대한 이야기를 진행했다.

1차 과제의 요구사항을 간략하게 정리해보자

과제의 기한과 요구사항은 다음과 같았다.

  • 기한: 21/11/1 16:00 ~ 11/3 10:00 (약 1.5일)
  • 개발 요구사항:
    • DB: mongodb 사용
    • 원티드 지원 과제 내용 포함(게시판 CRUD API)
    • 게시글 카테고리
    • 게시글 검색
    • 대댓글(1depth)
      • 대댓글 pagination
    • 게시글 읽힘 수
      • 같은 User가 게시글을 읽는 경우 count 수 증가하면 안 됨
    • Rest API 설계
    • 유닛테스트 구현
    • 1000만건 이상의 데이터를 넣고 성능테스트 진행 결과 필요
어떻게 과제를 해나갔는가

기존 선발과제를 확장해 나가는 방식이다 보니 우리는 한 팀원의 코드를 선택해 모델링 및 스웨거 기능까지의 환경을 맞추고 이를 기반으로 새로운 기능을 각 팀별로 추가해나가기로 했다. 이 과정에서 다른 팀원들의 게시판 CRUD API 코드를 간단하게 리뷰했고, 같은 기능이라도 다양한 방식의 코드 스타일이 있다는 것을 느꼈다. 그리고 새로운 기능을 구현하는 부분은 페어 프로그래밍으로 진행했다.

어려웠던 점은?

일단 첫 과제라 시간분배에 어려움이 컸다. 페어 프로그래밍으로 진행하다 보니 시간이 더 오래걸렸고, 대댓글 기능을 구현할 때 생각보다 시간이 많이 지체되었다. post 함수에서 댓글과 대댓글 로직을 같이 작성하다 보니 원하는 댓글에 대댓글이 추가되지 않는 문제가 있었는데, 이 부분을 조건절로만 해결하려고 했다. 대댓글을 구현할 때는 댓글 아이디를 패스 파라미터로 전달하여 로직을 따로 나눴어야 했는데 이 부분에서 한 가지 방법만 고집하였던게 이 기능을 어렵게 만들었던것 같다.

무엇을 느끼고 무엇을 배웠나

한 기능에 2~3시간을 투자해도 해결하는 아이디어를 찾지 못할 때는 일단 멈추고 다른 기능을 구현하는 것이 좋다는걸 배웠다. 우리는 대댓글 구현이 될듯 말듯한 느낌이 들어서 자꾸 시간을 지체했고, 결과적으로 기업에서 요구한 사항을 다 구현하지 못한채 과제를 제출하게 되었다. 당시에는 문제를 해결해 가는 과정이 더 중요하다고 생각했지만 끝나고 보니 미완성으로 제출한 부분이 많이 아쉬웠다.

2차 과제는 어떻게 하고 싶은가

팀원들과 업무 및 시간 분배를 잘 해서 기업의 요구사항을 다 충족해서 제출하는 것이 목표이다.
