에러 화면
이게 머선일..! 요건 구글에서 가져온 이미지이다. 나같은 경우는 develop 브랜치를 딴 후, default 브랜치인 main 브랜치에 풀리퀘를 보내려고 했을 때 이런 화면이 떴었다.
위와 같은 에러가 생긴 과정

보통은 새로운 레포를 파고 해당 레포를 클론하여 프로젝트를 진행했었다.
그런데 이번에는 이미 만들어진 다른 팀원의 레포를 git clone 해서 로컬에 프로젝트를 판 상황이었다. 기존에 있던 git log 내역을 지우고 새로 만든 레포로 리모트 설정을 해주면 되겠다 싶어서, .git을 지우고 git init을 하고 리모트 설정을 했다.
README.md만 생성한 채 main 브랜치에 푸시를 했고, develop 브랜치를 따서 과제가 끝나면 마지막에 main 브랜치로 풀리퀘를 보낼 생각이었다.
분명히 main과 develop의 변경사항이 다르기 때문에 풀리퀘를 보낼 수 있을 것이라고 생각했는데, 두 브랜치의 커밋 히스토리가 다르다는 이유로 보낼수가 없었다.

에러의 원인

에러메시지로 검색을 해보니, git과 연결된 폴더의 파일을 싹 바꿔버린 경우, main브랜치와 새로 바뀔 파일들이 같은 히스토리를 사용하지 않는다. 즉, 부모가 다르기 때문에 비교가 안된다는 것이었다. 또한 처음에 다른 팀원의 레포에서 git clone을 한 후 remote를 해제하지 않았고, git init을 하면 master 브랜치가 default 브랜치로 생성되는데 main으로 이름을 바꾸지 않은것도 문제가 되었던것 같다.

해결 방법

이럴 때는 덮어버릴 브랜치(나같은 경우 main)로 와서 –allow-unrelated-histories 플래그를 써서 강제로 develop 브랜치를 pull 하는 방법이 있다.

충돌이 나는 부분은 해결하고, 푸시를 하면 된다!
