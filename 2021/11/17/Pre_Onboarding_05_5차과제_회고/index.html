5차 과제의 요구사항을 간략하게 정리해보자

과제의 기한과 요구사항은 다음과 같았다.

  • 기한: 21/11/15 16:00 ~ 11/17 10:00 (약 1.5일)
  • 개발 요구사항:
    • DB: SQLite
    • REST API 기능
      • 임상정보를 수집하는 batch task
      • 수집한 임상정보에 대한 API
      • 수집한 임상정보 리스트 API
        • 최근 일주일내에 업데이트된 임상 정보 리스트
        • pagination 기능
    • ORM 필수 사용
    • 가산점 부분
      • Test 구현
      • 배포하여 웹에서 사용할 수 있도록 제공
      • 임상정보 검색 API
어떻게 과제를 해나갔는가
  • 첫날은 우리의 루틴대로
  • 기능 분배해서 둘째날부터 구현, 나는 수집한 임상정보에 대한 API
  • git rebase, squash 전략으로 협업
어려웠던 점은?
  • PR에 대해 코드리뷰를 받고, 내 코드 한줄 한줄에 대해 사용 이유를 정확하게 얘기할 수 있어야 함
무엇을 느끼고 무엇을 배웠나
  • 코드 한줄의 명확한 의미, 불필요한 코드는 작성할 필요 없음
  • 내 코드가 merge 되기까지 오래 걸림, 더 꼼꼼하게 코드 작성 필요
