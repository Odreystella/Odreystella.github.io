Django ORM

  • Object Relational Mapping(객체-관계 매핑)

  • 객체와 관계형 데이터베이스의 데이터를 매핑해주는 기술

  • 장점

    • SQL query가 아닌 직관적인 코드로 데이터를 조회, 추가, 수정, 삭제할 수 있어 편리하다.
    • DBMS에 대한 종속성이 적다 -> 데이터베이스를 바꾸더라도 ORM 코드를 그대로 사용할 수 있다.
    • 가독성 및 유지보수에 좋다.

  • 단점

    • 잘못된 사용으로 성능 저하, N+1 문제 등을 야기할 수 있다.
    • SQL query를 작성하지 않다 보니 정확한 원리를 이해하기 어렵다.
QuerySet 이란?
  • Query란 데이터베이스에 정보를 요청하는 질의문이며, 파이썬으로 작성한 코드가 SQL로 매핑되어 QuerySet라는 자료 형태로 값이 넘어온다. 즉, ORM의 결과로 전달받은 객체들의 목록이다.
  • 1개 이상의 데이터를 불러오기 때문에 필터를 걸거나 정렬을 사용할 수 있다.
  • 1개의 Query(메인 쿼리)와 0~N개의 QuerySet(추가 쿼리셋)으로 구성된다
QuerySet을 통해 알아보는 Django ORM 특징

  1. Lazy Loading(지연 호출)

    • 1-1. 정말 필요해야만 SQL을 호출한다.
      • Tire QuerySet을 선언하고 사용하지 않아서 SQL이 호출되지 않는다.
    • 1-2. 정말 필요한 시점에만 SQL을 호출한다.
      • .count()등 실제로 QuerySet에 담겨있는 데이터를 이용할 때 SQL이 호출된다.
    • 1-3. 정말 필요한 만큼만 호출한다.
      • 하나의 tire 객체만 얻기 위해 LIMIT 1 옵션이 걸린 SQL을 호출한다. 하지만 후에 tires QuerySet에 있는 데이터를 이용할 때 다시 SQL을 호출하기 때문에 두 번 호출하지 않도록 주의해야 한다. 해결방법으로 즉시 로딩을 사용한다.

  2. Caching(캐싱)

    • QuerySet은 호출한 SQL에 대한 결과를 캐싱하여 저장하는 특성이 있다.
    • list()을 사용하여 먼저 캐싱한 후, 데이터를 조작하면 SQL이 호출되지 않는다.

  3. Eager Loading(즉시 호출)

    • QuerySet는 기본적으로 Lazing Loading 전략을 이용하는데, 한번에 여러 개의 데이터를 가져오고 싶을 때, select_related(), prefetch_related() 메소드를 사용하여 Eager Loading 전략을 사용할 수 있다.
    • Eager Loading 전략을 사용하지 않으면 N+1 문제를 야기할 수 있다.
      • N+1 문제: Lazing Loading 전략 때문에 발생한다. 예를 들어 Tire 모델과 Trim 모델이 1:N 관계 일때, Tire 객체에서 Trim 객체의 데이터에 접근하면 처음에 모든 Tire를 조회하기 위한 SQL이 1번 수행되고, 이 안에서 Trim의 id를 조회하기 위해 Tire의 객체의 갯수만큼 쿼리가 날아간다. N+1개의 쿼리가 발생하는 이유는 처음에 Tire의 데이터를 SQL로 한 번에 가져왔어도 trim_id의 데이터를 가져오지 않았기 때문에 N번의 쿼리가 호출되기 때문이다.
