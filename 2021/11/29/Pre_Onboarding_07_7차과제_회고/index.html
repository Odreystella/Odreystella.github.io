7차 과제의 요구사항을 간략하게 정리해보자

과제의 기한과 요구사항은 다음과 같았다.

  • 기한: 21/11/22 16:00 ~ 11/29 10:00 (약 7일)
  • 개발 요구사항:
    • 선호 기술 스택: Express/Nest JS or Spring Framework
    • DB: RDBMS 사용
    • REST API 기능
      • 사용자 생성 API
        • ID/Password로 사용자 생성
        • 인증 토큰 발급
      • 사용자가 소유한 타이어 정보를 저장하는 API
        • 인증된 사용자만 호출
        • 한번에 최대 5명까지의 사용자에 대한 요청이 가능하게 구현
      • 사용자가 소유한 타이어 정보 조회 API
    • ORM 사용
    • 제시한 응답 코드 사용
    • 가능하다면 docker 사용
어떻게 과제를 해나갔는가

팀 과제때 했던 루틴대로 둘째날까지는 초기환경 셋팅, 프로젝트 이슈 작성, 모델링, 마이그레이션을 진행했다. 개인 과제여도 git rebase 브랜치 병합 전략에 조금 더 익숙해지기 위해 git rebase 전략으로 코드를 병합했다. PR을 올려도 코드를 리뷰해줄 팀원들이 없어서 외로웠지만 한편으론 편하긴 했다. 그래도 코드리뷰를 받으면서 새롭게 알게되는 부분들이 훨씬 많아서 힘들긴해도 팀플이 더 좋긴 하다ㅋㅋ

Framework와 DB는 익숙한 Django와 in memory인 SQLite을 사용하기로 결정했고, 구현해야 할 REST API는 총 3개여서 사용자 생성 API부터 개발하기 시작했다. PR은 단위테스트를 진행 한 후에 보냈다.

어려웠던 점은?

여전히 모델링은 어렵다…몇년차 개발자가 되어야 모델링을 잘 짠다고 자신있게 말할 수 있게될 지 궁금하다 ㅎㅎ 사용자가 소유한 타이어 정보를 저장하는 API 때문에 실제 카닥의 API를 호출해서 데이터를 받았는데, 이 내용을 보고 테이블과 필드명을 짰다. 가장 고민이 되었던 부분은 타이어가 앞, 뒤 두개인데 정규화를 해서 테이블을 나눌지 하나의 테이블로 만들고 name이라는 필드로 앞, 뒤를 구분할지였다. 결국엔 하나의 테이블로 만들긴 했는데 실무에서는 어떻게 설계했는지 궁금하다.

두번째는 사용자가 소유한 타이어 정보를 저장하는 API를 만들 때, 코드가 너무 길어졌는데 이 부분을 어떻게 재사용 가능한 코드로 쪼갤지가 어려웠다. 구현할 때는 크게 생각을 못했는데 한참 지나고 보니 코드가 지저분해 보였다. 외부 API로부터 데이터를 받아오는 부분을 따로 빼도 괜찮을 것 같다는 생각이 드는데 나중에 리팩토링 해봐야겠다.

무엇을 느끼고 무엇을 배웠나

일단 API를 만드는 일은 재밌다. 200코드를 받았을 때의 그 쾌감이란…ㅎㅎ 물론 잘 안될때는 힘들지만 재미를 느끼는 부분이 더 크다. 과제가 끝나고 다른 팀원과 얘기를 하면서 알게 되었는데 외부 API에 의존하는 API일 경우 단위테스트 할 때 Mocking 기법을 사용해야 한다는 것이었다. 과제를 제출할 당시에는 시간이 없어서 수정하지 못했지만 팀원이 참고했던 자료를 공유해줘서 리팩토링 해봐야될 것 같다. (실제로 Mocking을 공부하고 리팩토링 했다!)

마지막 과제 또는 프리온보딩을 무사히 마친 소감

내 생애 가장 길었던 한달이 아니었나 싶을 정도로 팀 과제를 할때는 시간 압박과 온라인 상에서의 협업으로 정신적, 체력적으로 많이 힘들었었다. 이런게 협업이구나 싶을 정도로 우리가 만든 컨벤션을 잘 지키고 그 외적으로 커밋 메시지, PR 메시지, 에러 메시지 등 신경써야 하는 부분들이 많다는 것을 배웠다. 협업 경험이 가장 필요하다고 생각했을 시점에 떨어질거라고 생각했던 프리온보딩 선발과제가 통과되서 이 기회를 잘 활용해야 겠다는 마음가짐으로 참여했는데, 힘들어도 포기하지 않고 마칠 수 있어서 스스로가 뿌듯했다. 이 코스에 참여할 수 있어서 정말 정말 감사했다.

이 경험으로 배운 점은 정말 많다. 열거해 보자면..

  • 자신감이 생겼다.
  • 온라인 환경에서 협업하는 것에 익숙해졌다.
  • Django로 REST API를 구현하는데 익숙해졌다.
  • Unit Test, Mocking을 활용해서 테스트 코드를 작성하는데 익숙해졌다.
  • git 브랜치 병합 전략으로 Rebase, Squash를 할 수있다.
  • PR을 올려서 코드리뷰를 경험해봤다.
  • AWS EC2를 사용하여 7번의 배포 경험이 있다.
  • Postman, Swagger를 사용해서 API를 테스트하고 문서를 만드는데 익숙해졌다.
  • 새로운 프레임워크 flask, 새로운 ORM Sqlalchemy를 사용해보면서 새로운 기술에 대해 두려움이 줄었고, 공식 문서 보는데 익숙해졌다.
  • 사람들과 소통하는 방법, 의견을 명확하게 전달해야 하는 것을 배웠다.

이 정도로 정리할 수 있을 것 같다. 소중한 경험을 바탕으로 얼른 현업에서 멋진 개발자로 일하고 싶다:D
