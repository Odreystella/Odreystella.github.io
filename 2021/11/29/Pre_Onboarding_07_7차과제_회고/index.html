7차 과제의 요구사항을 간략하게 정리해보자

과제의 기한과 요구사항은 다음과 같았다.

  • 기한: 21/11/22 16:00 ~ 11/29 10:00 (약 7일)
  • 개발 요구사항:
    • 선호 기술 스택: Express/Nest JS or Spring Framework
    • DB: RDBMS 사용
    • REST API 기능
      • 사용자 생성 API
        • ID/Password로 사용자 생성
        • 인증 토큰 발급
      • 사용자가 소유한 타이어 정보를 저장하는 API
        • 인증된 사용자만 호출
        • 한번에 최대 5명까지의 사용자에 대한 요청이 가능하게 구현
      • 사용자가 소유한 타이어 정보 조회 API
    • ORM 사용
    • 제시한 응답 코드 사용
    • 가능하다면 docker 사용
어떻게 과제를 해나갔는가
  • 팀과제의 루틴대로 초기환경 셋팅, 프로젝트 이슈 작성, 모델링, 마이그레이션 먼저
  • 개인과제여도 branch 전략으로 진행
  • 사용자 생성 API부터 구현, 하나의 기능 -> 테스트코드 까지 작성하여 PR
어려웠던 점은?
  • 요구사항 대로 모델링 짜는 부분
  • 카닥의 API 호출해서 사용하는 부분, 테스트하는 방법 - 테스트 모킹
무엇을 느끼고 무엇을 배웠나
  • 재밌다
마지막 과제를 하고 난 소감
  • 협업 경험 -> 사람들과 소통하는 방법, 의도를 명확하게 전달해야 함을 배움
  • 온라인 환경에서 협업하는 것 익숙
  • git 병합 전략 Rebase, Squash에 익숙
  • 유닛테스트 코드 작성에 익숙
  • 배포 경험
  • API 문서 작성(postman, swagger)
  • PR 올리면 코드리뷰
  • 새로운 프레임워크flask, 새로운 ORM sqlalchemy 경험
  • django로 rest api 구현 익숙
