테스트 모킹 개념
  • 모킹(Mocking): 외부에 의존하는 부분을 임의의 가짜로 대체하는 기법
  • 데이터베이스 또는 외부 API에 의존하지 않고 독립적으로 실행이 가능한 단위 테스트를 작성하기 위해 사용되는 테스팅 기법이다.
  • 데이터베이스 또는 외부 API에 의존하는 코드를 테스트할 때 사용한다.
Mock 객체 및 제공되는 메서드와 속성들

  • 파이썬 3.3부터 언어 자체에 기본 내장되어 있는 모킹 라이브러리에서 mock 모듈을 임포트하여, 코드의 특정 부분을 mock 객체로 대체하고 어떻게 사용되었는지 검증할 수 있다.

    • Mock 객체를 알아보자
      • mock 객체가 어떻게 작동할 지 지정해줄 수 있다.
    • Mock 객체의 메서드와 속성들을 알아보자
      • 자신에게 발생했던 작업들을 검증할 수 있게 도와주는 메서드와 속성이다.
Magic Mock
  • 파이썬에는 매직 메서드라는 개념이 있는데, 기본적으로 Mock 클래스를 사용하면 이러한 매직 메서드가 자동으로 모킹되지 않는다.
    • 매직 메서드는 __str__, __init__과 같이 특수한 목적으로 쓰이는 메서드들을 정의한다.
  • Mock 클래스의 확장 버전인 MagicMock클래스를 사용하면 매직 메서드를 미리 알아서 모킹해두기 때문에 편리하다.
Django에서 테스트 모킹 방법

  • @patch() 데코레이터 사용

    • 특정 모듈의 함수나 클래스를 patch의 첫번째 인자로 받아 patching 하면 이는 MagicMock 객체로 교체된다.
    • patching을 하면 Mock 객체를 테스트 메서드의 매개 변수로 추가하게 되는데, MagicMock으로 교체된 Mock 객체가 할당된다.
    • 특정 범위 내에서만 patching이 가능하도록 하여 필요한 단위테스트 메서드 위에 선언해준다.
    • 해당 Mock의 객체에 return_value나 status_code 속성을 지정하여 예상한 결과가 나오는지 검증한다.
    • 실제 프로젝트에서 단위 테스트를 작성할 때는 mock = Mock()처럼 직접 mock를 생성하는 것보다 @patch()를 사용하는 것이 일반적이다.

  • 프리온보딩코스의 개인과제였던 카닥 단위 테스트에 테스트 모킹을 적용한 코드

    • 기존의 단위 테스트에서 달라진 점은 크게 4 부분이다.
    • # 1 @patch("cars.views.requests")
      • 데코레이터를 선언하여 views.py 의 post()함수 내에 존재하는 cars.views.requests 패키지를 patching한다.
      • cars.views.requests 를 MagicMock 객체로 교환하기 때문에 실제 네트워크 연동이 일어나지 않는다.
    • # 2 def test_post_tire_info_success(self, mock_request):
      • patching을 하면 mock 객체가 테스트 메서드의 인자로 mock_request 추가되는데, 여기에 MagicMock 객체가 할당된다.
      • 즉, cars.views.requests가 MagicMock 객체로 교체된다.
      • mock_request을 찍어보면 <MagicMock name='requests' id='140239489664960'> MagicMock이 할당된 것을 알 수 있다.
    • # 3 16 ~ 35번 코드까지
      • 실제로 외부 API를 호출했을 때 리턴될 json형태의 데이터들을 json 메서드의 리턴 객체에 임의로 지정한다.
    • # 4 mock_request.get = MagicMock(return_value=TireInfoMockResponse())
      • requests.get() 요청을 할 때 MagicMock의 return_value에 임의로 만든 TireInfoMockResponse()를 지정해준다.
      • views.py에서 response를 찍어보면 TireInfoResponse에 임의로 만들어준 리턴값이 나와 reponse = TireInfoResponse.json()으로 지정해준 리턴값으로 변경된 것을 알 수 있다.
