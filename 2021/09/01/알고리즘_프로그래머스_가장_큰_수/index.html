    문제 설명

    0 또는 양의 정수가 주어졌을 때, 정수를 이어 붙여 만들 수 있는 가장 큰 수를 알아내 주세요.

    예를 들어, 주어진 정수가 [6, 10, 2]라면 [6102, 6210, 1062, 1026, 2610, 2106]를 만들 수 있고, 이중 가장 큰 수는 6210입니다.

    0 또는 양의 정수가 담긴 배열 numbers가 매개변수로 주어질 때, 순서를 재배치하여 만들 수 있는 가장 큰 수를 문자열로 바꾸어 return 하도록 solution 함수를 작성해주세요.

    제한사항
    • numbers의 길이는 1 이상 100,000 이하입니다.
    • numbers의 원소는 0 이상 1,000 이하입니다.
    • 정답이 너무 클 수 있으니 문자열로 바꾸어 return 합니다.
    입출력 예
    numbers return
    [6, 10, 2] “6210”
    [3, 30, 34, 5, 9,] “9534330”
    [0, 0, 0, 0] “0”

    코드 해석
    1. numbers는 숫자가 담긴 리스트인데, 문자를 정렬하는게 필요해서 map를 사용하여 문자열이 담긴 리스트로 재정의해준다. 숫자를 그대로 정렬하면 원하는 결과가 안나온다..
    2. 내가 원하는 규칙대로 정렬하는 방법으로 sort()메서드의 key 파라미터를 활용할 수 있는데, 함수(빌트인함수 or 사용자 정의함수)를 넣어준다.
      [“3”, “34”, “32”] 를 큰 숫자부터 정렬한다고 했을 때, 여기서 나올 수 있는 가장 큰 수는 “34332”이다. 그냥 비교하게 되면 “34323”이 나오기 때문에 [“333”, “343434”, “323232”]문자열의 길이를 3배로 늘린 후 비교해야 한다. 제한사항에 numbers의 원소는 0이상 1,000이하라고 하여 길이를 3배 늘려주는 것이다. 정확히는 모르겠는데 파이썬의 정렬 알고리즘이 앞의 숫자부터 차례로 비교하며 정렬한다고 한다. reverse=True의 의미는 내림차순.
    3. 내림차순으로 잘 정렬된 문자열들을 join()메서드를 사용해서 붙여준다. 여기서 “0000”이 리턴되는 케이스를 고려해 문자열 -> 숫자 -> 문자열로 바꿔주는 코드도 있는데 난 따로 조건을 주는게 더 가독성이 좋아서 다른 코드를 가져왔다.
    4. “0000”이 리턴되는 경우에 통과가 안되기 때문에 “0”으로 만들어주는 조건을 준다.

    배운 점

    • sort(), sorted()는 key, reverse라는 파라미터를 갖고 있다.
      key : 함수가 들어가고 내가 정의한 규칙대로 정렬이 되는 아주 대단한 파라미터다. 문자열 같은 경우 lower, upper, len 등 빌트인 함수로 정렬도 가능하다. 다중 정렬도 가능한데 아래 출처를 참고하자.
      reverse : bool 값이 들어가고 False가 디폴트(오름차순), True는 내림차순을 의미한다.
