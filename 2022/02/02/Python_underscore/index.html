무시하는 값

  • for 문에도 응용할 수 있다.

네이밍
  1. variable: public 변수
    • 일반적으로 사용하는 변수를 의미한다.
  2. _variable (앞에 언더바): protected 변수
    • 해당 모듈내에서만 이 변수/함수를 사용하겠다는 의미이다.
    • 협업할 때 다른 팀원에게 ‘이 변수/함수는 이 모듈 내부에서만 사용할 거다’ 라고 명시적으로 힌트를 줄 수 있다.
    • 파이썬은 private와 public의 확실한 구분이 없어 완전한 의미의 private는 아니기 때문에 여전히 해당 변수/함수에 접근하거나 사용할 수 있다.
    • 외부 모듈에서 해당 모듈을 from module_name import * 식으로 전체 임포트할 때 앞에 _가 붙은 변수/함수는 임포트 하지 않는다.
  3. variable_ ( 뒤에 언더바)
    • 파이썬 키워드와 변수/함수명의 충돌을 피하기 위해서 사용한다.
    • 예를 들어, list나 print 같은 키워드를 변수/함수명으로 사용하고 싶을 때 list_, print_와 같이 사용하면 된다.
  4. __variable (앞에 언더바 두개): private 변수
    • 네이밍 룰이라기 보다 문법적인 요소로 이해해야 한다.
    • 네임 맹글링(name mangling) 을 위한 경우인데, 파이썬이 해당 변수/함수의 이름을 짓이겨서 바꿔버리는 것을 의미한다.
    • 맹글링(짓이기다라는 의미)을 당한 변수/함수는 본연의 이름으로 접근할 수 없게 된다. 즉 접근이 안되기 때문에 보호가 된다. 그러나 수정은 가능하다
    • 맹글링 당한 변수/함수는 _클래스명__변수/함수명 형태로 이름이 변경된다
  5. __ variable __ (앞뒤로 언더바 두개씩)
    • 파이썬이 특별히 정의한 변수나 함수에 사용되는 네이밍 룰이다.
    • 이 네이밍 룰을 사용해서 직접 함수를 만드는 일은 거의 없고, 대부분 오버라이딩 할 때 사용하게 된다.
    • 이 룰이 적용된 함수는 매직 메소드(Magic Method)나 던더 메소드(Dunder Method, Double Underscore Method)라고 부른다.
    • 대표적으로 __init__ 함수가 있다. 클래스의 인스턴스가 생성될 때 가장 처음으로 실행되는 함수로 생성자 역할을 한다.
네임 맹글링 사용하는 이유

  1. 클래스의 속성값을 외부에서 접근하기 힘들게 할 때 (private화)

    • 완벽한 private라기보다 외부의 접근을 조금 어렵게 숨기는 역할

  2. 하위 클래스가 상위 클래스의 속성을 오버라이딩 하는 것을 막을 때

    • 클래스가 확장되면서 속성 이름간의 충돌을 막거나, 특히 오버라이딩을 막고 싶을 때 맹글링을 적용할 수 있다.
    • 맹글링을 하면 이름이 바뀌기 때문에 오버라이딩 할 수 없다.
