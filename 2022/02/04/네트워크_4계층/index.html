4계층에서 하는 일
  • 전송 계층(Transport layer)은 송신자의 프로세스와 수신자의 프로세스를 연결하는 통신 서비스를 제공한다.
  • 전송 계층은 연결 지향 데이터 스트림 지원, 신뢰성, 흐름 제어, 다중화와 같은 편리한 서비스를 제공한다.
4계층에서 사용하는 주소, 포트 번호
  • 특정 프로세스가 사용하는 주소
  • 내 컴퓨터 기준으로 하나의 포트는 하나의 프로세스만 사용할 수 있다. 하나의 포트번호로 다른 컴퓨터들의 여러 요청이 들어오는 것은 가능하다.
  • 특정 프로세스와 특정 프로세스가 통신하기 위해 사용한다.
  • 포트 번호는 일반적으로 정해져 잇지만 무조건 지켜야 하는 것은 아니다.
  • 예를 들어 일반적으로 웹 서비스는 80번 포트를 사용하지만 다른 포트번호를 사용해도 된다.
  • 3가지로 구분될 수 있다.
    • Well-Known 포트: 잘 알려진 포트 번호이다.
      서비스 이름 포트 번호
      FTP 20, 21번
      SSH 22번
      TELNET 23번
      DNS 53번
      HTTP 80번
      HTTPS 443번
    • Registered 포트: 유명한 서비스들이 등록해둔 포트 번호이다.
      서비스 이름 포트 번호
      오라클 DB 1521번
      MySQL DB 3306번
      PostgreSQL DB 5432번
    • Dynamic 포트: 49152번 ~ 65535번 사이에서 일반 사용자들이 사용한다.
4계층 프로토콜 TCP, UDP
  1. TCP
    • 안전한 연결을 지향한다. 연결이 된 것을 확인한 후에 데이터를 전송한다.
  2. UDP
    • 비연결 지향으로 빠른 연결을 지향한다. 연결 여부와 상관없이 데이터를 전송한다.
