3계층에서 하는 일
  • 라우터 라는 장비를 통해 서로 다른 LAN 대역을 연결시켜준다.
  • 멀리 있는 곳과 통신하기 위해서는 3계층이 필요하다.
3계층에서 사용하는 주소, IP 주소

  • 10진수로 표기하고, 4Byte를 사용한다.

  • 4개의 필드가 . 으로 구분되어 있고, 하나의 필드에는 0~255까지의 숫자만 올 수 있다.

    • 예) 0.0.0.0 ~ 255.255.255.255

  • 초기에는 A~E 클래스로 네트워크를 구분하였는데, 낭비가 심한 단점이 있었다.

    • A클래스 0. 0.0.0 ~ 127. 255.255.255 첫 번째 필드는 네트워크를 구분, 나머지 필드는 PC를 구분하는데 사용
    • 일반적으로 C클래스를 많이 썼음

  • 이를 보완하기 위해 Classless IP를 사용하고, 서브넷 마스크(IP 주소에 대한 네트워크 대역을 규정)와 함께 사용한다.

  • 현재는 그래도 IP가 부족하여 사설 IP와 공인 IP를 도입하여 사용한다. IPv6도 사용한다.

    • 공인 IP: 실제 네트워크 통신망(외부)과 통신할 때 사용하는 IP 주소. 즉, 다른 네트워크 대역으로 갈 때 사설 IP가 공인 IP로 바뀜
      • 네이버에 내 ip 주소 확인하면 나오는 공인 IP 211.207.75.23
      • 실제 인터넷 세상에서는 공인 IP로만 통신한다.
      • 외부에서 볼 때는 사설 IP대역이 없는 것처럼 보인다. 예를 들어 하나의 공유기에 30대의 PC가 묶여 있는 네트워크는 외부에서 볼 때는 하나의 공유기로 본다. 사설 IP는 공유기만 알고 있다. 한 PC가 어떤 요청을 할 때 공유기는 NAT table에 기록하고 응답을 받으면 전달해준다. 요청하지 않았는데 응답을 받았다면 전달되지 않는다.
      • 공인 IP 1개당 2^32(0.0.0.0 ~ 255.255.255.255)개의 사설 IP를 가질 수 있다.
    • 사설 IP: 같은 네크워크 대역에서 사용하는 IP 주소
      • 실제 내 IP는 192.168.35.172
    • NAT(Network Address Translation): 특정 IP를 특정 IP로 바꿔주는 기술이다.

  • 특수한 IP 주소

    1. 0.0.0.0/0: 나머지 모든 IP를 의미한다.
    2. 127.0.0.1: localhost(나 자신을 나타내는 주소)
    3. 게이트웨이 주소: 외부와 통신할 때 나가는 게이트를 알려줌(네트워크 대역에서 사용할 수 있는 가장 작거나 큰 주소, 일반적으로 공유기의 IP를 사용)
3계층 프로토콜
  1. ARP
    • IP 주소를 이용해 MAC 주소를 알아온다.
  2. IPv4
    • WAN에서 통신할 때 사용한다.
  3. ICMP
    • 서로가 통신되는지 확인할 때 사용한다.
