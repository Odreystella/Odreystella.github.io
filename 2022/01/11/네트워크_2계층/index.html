2계층에서 하는 일
  • 하나의 네트워크 대역(LAN), 즉 같은 네트워크 상에 존재하는 여러 장비들 중에서 어떤 장비가 어떤 장비에게 데이터를 전달한다
  • 오류제어(보내는 데이터에 오류가 있는지), 흐름제어(누가 누구에게 데이터를 보내는지)도 수행한다.
  • 2계층은 LAN에서만 통신할 때 사용한다.
  • 다른 네트워크와 통신할 때는 항상 3계층(3계층 주소와 3계층의 프로토콜) 이 도와줘야 한다.
2계층에서 사용하는 주소, MAC 주소
  • LAN에서 통신할 때 사용하는 주소 체계이다.
  • 물리적인 장비에 부여된 주소값이기 때문에 물리적인 주소라고 부른다.
  • 16진수로 표현한다.
    • 예) 6C:29:95:04:EB:A1 (6Byte)
    • 6C-29-95: OUI(IEEE에서 부여하는 일종의 제조회사 식별 ID)
    • 04-EB-A1: 고유번호(제조사에서 부여한 고유번호)
  • 전세계에 있는 네트워크 장비들은 서로 다른 주소값을 가지고 있고, 쉽게 바꿀 수 없다.
2계층 프로토콜, Ethernet
  • LAN에서 통신할 때 사용하는 프로토콜이다.
  • 14Byte를 사용한다.
  • Destination Address(6Byte) 에 목적지 MAC 주소가 위치한다.
  • Source Address(6Byte) 에 출발지 MAC 주소가 위치한다.
  • Ethernet Type(2Byte) 에 페이로드에 상위 프로토콜의 타입(3계층 어떤 프로토콜)이 캡슐화되어 있는지 알려준다.
    • 전기적 신호로 패킷이 전달되서 받는 쪽에서는 목적지가 맞는지 확인한다. 맞으면 데이터를 확인해봐야 하는데 Ethernet Type으로 상위 프로토콜이 뭔지(ARP인지 IPv4인지) 알 수 있어야 분석할 수 있다. 데이터는 2진수(0101 블라블라)로 시작하기 때문에 Ethernet Type이 없으면 분석할 수 없다.
    • 08 00이 오면 IPv4 프로토콜을 의미한다.
    • 08 06이 오면 ARP 프로토콜을 의미한다.
실습
  1. 내 컴퓨터 MAC 주소 확인하기
    • Ubuntu에서는 ifconfig -a | grep eth 커맨드로 MAC 주소 확인한다.
    • 내거는 OUI가 dc:71:96 으로 Intel Corp에서 만들었다.
  2. Ethernet 프로토콜 캡쳐 및 분석하기
    • Wireshark를 실행하고, 터미널에 ping 8.8.8.8(구글 DNS) 입력하여 통신해보자
    • ICMP 프로토콜인 것을 클릭하여 패킷을 확인해보면 Ethernet 프로토콜은 아래와 같이 14Byte 인 것을 확인할 수 있다.
      • 00 23 aa d8 e4 bd dc 71 96 f7 c9 1d 08 00
      • 목적지 MAC 주소 출발지 MAC 주소 Ethernet Type
      • ICMP는 IP 프로토콜의 단점(전송상태에 대한 관리가 이루어지지 않아 신뢰할 수 없음)을 보완하는 프로토콜이다. IP 패킷 전송 중 에러 발생 시 에러 발생 원인을 알려주거나 네트워크 상태를 진단해주는 기능을 제공한다. 통신 유무 확인시 사용하는 ping 명령이 사용하는 프로토콜이다.
