개념 정의
  • 노드(네트워크에 속한 컴퓨터 또는 통신장비)들이 데이터를 공유할 수 있게 하는 디지털 전기 통신망의 하나이다.
  • 분산되어 있는 컴퓨터를 통신망으로 연결한 것이다.
  • 네트워크에서 여러 장치둘은 노드 간 연결을 사용하여 서로에게 데이터를 교환한다
인터넷
  • 문서, 그림, 영상과 같은 여러가지 데이터를 공유하도록 구성된 세상에서 가장 큰 네트워크
  • cf) www: 인터넷을 통해 웹과 관련된 데이터를 공유하는 것
네트워크의 분류
  1. 크기에 따라
    • LAN(Local Area Network): 가까운 지역을 하나로 연결한 네트워크
      • ex) 한 강의장 내에 하나의 장비에 여러대의 컴퓨터가 연결된 것
    • WAN(Wide Area Network): LAN과 LAN을 다시 하나로 연결한 네트워크
  2. 연결 형태에 따라
    • Star형: 중앙 장비(하나의)에 모든 노드가 연결되어 있는 네트워크, LAN 대역으로 연결할 때 주로 사용하는 방식
      • 단점: 하나의 장비에 연결되어 있기 때문에 장비가 고장나면 모든 장비의 연결이 끊어진다
    • Mesh형: 여러 노드들이 서로 그물처럼 연결되어 있는 네트워크, WAN 대역으로 연결할 때 주로 사용하는 방식
      • 예) 해저 케이블로 전 세계가 연결되어 있는 인터넷
      • 장점: 하나의 네트워크 장비가 망가져도 다른 장비를 통해 우회해서 사용할 수 있다
    • 혼합형: LAN과 WAN이 혼합된 형태
네트워크의 통신 방식
  • 유니 캐스트: 같은 네트워크 대역에 있는 특정 대상이랑만 1:1로 통신한다.
  • 멀티 캐스트: 같은 네트워크 대역에서 여러명의 사용자가 있었을 때, 특정한 다수랑 통신한다.
  • 브로드 캐스트: 같은 네트워크 대역에 있는 모두랑 통신한다.
프로토콜
  • 통신을 하기 위한 일종의 양식, 약속
  • 네트워크에서 노드와 노드가 통신할 때, 어떤 노드가 어떤 노드에게 어떤 데이터를 어떻게 보내는지 작성하기 위한 양식
  • 네트워크 통신 방식에 따라서 프로토콜도 다름
  • 예) 가까운 곳과 통신할 때: Ethernet 프로토콜(MAC 주소)
  • 예) 멀리 있는 곳과 통신할 때: ICMP, IPv4, ARP(IP 주소)
  • 예) 여러가지 프로그램으로 통신할 때: TCP, UDP(포트 번호) 카카오톡으로 갈건지 크롬으로 갈건지
  • 실제 통신할 때는 [ Ethernet | IPv4 | TCP | Data ] 이런식으로 여러 프로토콜을 같이 사용하는데, 합쳐진 형태를 캡슐화라고 한다.
