네트워크 모델의 종류
  1. TCP/IP 모델
    • 1960년대 말 미 국방성의 연구에서 시작되어 1080년대 초 프로토콜 모델로 공개
  2. OSI 7계층 모델
    • 1984년 네트워크 통신을 체계적으로 다루는 ISO에서 표준으로 지정한 모델
    • 데이터를 주고받을 때 데이터 자체의 흐름을 각 구간별로 나눠 놓은 것

두 모델의 비교
  • 공통점
    • 네트워크를 계층별로 나누어 분류했다.
    • 각각의 계층별로 역할을 지정하여 사용했다.
  • 차이점
    • 계층의 수가 다르다.
    • OSI는 논리적으로 역할을 나누었고, TCP/IP는 실제 프로토콜 기반으로 나누었다.
    • OSI는 통신 전반에 대한 표준을 나타냈고, TCP/IP는 데이터 전송기술에 특화해서 보여준다.
패킷
  • 네트워크 상에서 전달되는 데이터를 통칭하는 단어이다.
  • 블록단위로 되어 있다. 실제로는 0과 1로 이루어진 전기 데이터인데 우리 눈에는 블록처럼 보인다.
  • 내부에는 여러 제어 정보(누가 누구에게 보내는지, 어떻게 어떤 데이터를 보내는지) 와 사용자의 데이터(=페이로드) 로 이루어져있다.
  • 여러가지 프로토콜로 캡슐화되어 있는데, 순서가 있는 조합이다. (하위 프로토콜 앞에 상위 프로토콜이 올 수 없다. 즉 | IPv4(3계층) | Ethernet(2계층) | TCP(4계층) | 데이터 | 의 조합은 만들 수 없다. 같은 계층의 프로토콜을 붙일 수는 있다.)
캡슐화
  • 페이로드에 프로토콜을 헤더로 붙이는 과정이다. 풋터(Ethernet만 사용)는 잘 사용하지 않는다.
  • 일반적으로 내가 누군가에게 패킷을 보낼 때 사용한다
  • 상위계층에서 하위계층(응용 계층→물리 계층)으로 내려갈 때 프로토콜을 붙인다.
  • 패킷을 이용한 통신 과정 예시
    1. 네이버 검색창에 코로나를 입력한다.
    2. 네이버의 웹 서버와 통신해야 하기 위해 헤더로 TCP를 캡슐화한다.
    3. 멀리 있는 곳과 통신해야 하니까 TCP+데이터를 페이로드로, 헤더에 IPv4를 캡슐화한다.
    4. 가까운 곳과 통신해야 하니까 IPv4+TCP+데이터를 페이로드로, 헤더에 Ethernet을 캡슐화한다.
역캡슐화
  • 패킷을 받았을 때 하위 프로토콜부터 하나씩 확인하면서 데이터를 확인하는 과정이다.
PDU(Protocol Data Unit)
  • 계층별 패킷의 형태를 부르는 이름
  • 3계층 PDU인 패킷과 네트워크 상에서 전달되는 데이터를 통칭하는 단어인 패킷은 다르다.
