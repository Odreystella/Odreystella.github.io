코딩테스트 합격자되기 파이썬편 - 리스트

리스트의 특징

  1. []를 사용해서 선언한다
  2. 데이터의 순서가 있는 시퀀스 타입이고, 변경 가능한 뮤터블 컬렉션이다.
    • 순서가 있기 때문에 배열처럼 인덱스로 접근이 가능하다.
  3. 내부는 동적배열로 구현되어 있다.
    • 배열의 크기를 내부에서 알아서 관리한다는 의미
    • id()로 주소값을 확인해 볼 수 있다.

리스트의 메서드와 시간복잡도

  • .append(item): O(1)
    • 반환값 없음
  • .insert(idx, item): O(N)
  • .pop(): O(1)
    • 마지막 원소 삭제
    • 삭제한 원소를 반환
  • .pop(idx): O(N)
    • 인자는 인덱스
    • 삭제한 원소를 반환
    • 리스트의 맨 앞에 원소를 삭제하는 경우에는 deque의 popleft()가 시간복잡도가 O(1)이라 이걸 쓰는게 좋음
  • .remove(item): O(N)
    • 맨 처음 나오는 item을 삭제
    • 반환값 없음
    • item이 리스트에 존재하지 않는 경우 ValueError
  • .extend(list): O(list) 추가할 리스트의 길이 만큼
    • 반환값 없음
  • in: O(N)
    • Boolean값 반환함
  • .index(item): O(N)
  • .sort(): O(NlogN)
  • .count(item): O(N)
  • .reverse(): O(N)
  • min(lst), max(lst): O(N)
  • lst[idx] : O(1)
  • len(lst): O(1)
  • lst[i:j] : O(k)
  • lst1 + lst2 : O(n+m) n은 lst1의 길이, m은 lst2의 길이

collections.deque의 메서드와 시간복잡도

  • .append(item): O(1)
    • 반환값 없음
    • deque 오른쪽(끝)에 item 추가
  • .appendleft(item): O(1)
    • deque 왼쪽(앞)에 item 추가
  • .pop(): O(1)
    • deque 오른쪽(끝) 원소 삭제
    • 삭제한 원소 반환
    • 원소가 없으면 IndexError
  • .popleft(): O(1)
    • deque 왼쪽(앞) 원소 삭제
    • 삭제한 원소 반환
    • 원소가 없으면 IndexError
  • .rotate(k): O(k)
    • k수만큼 회전하는데 양수면 오른쪽으로, 음수면 왼쪽으로 회전한다.

코딩테스트에서 리스트를 사용할 때 주의할 점

  • 맨 앞에 원소를 삽입하거나, 맨 앞에 원소를 삭제하는 경우
    • 시간복잡도가 O(N)이다.
    • collections에서 deque를 임포트해서 사용하는 것이 좋음. 시간복잡도가 O(1)이기 때문!
      • .appendleft(), .popleft()
  • 특정 원소가 있는지 자주 탐색이 필요한 경우
    • 배열이기 때문에 어떤 원소가 있는지 없는지 확인하려면 앞에서부터 탐색하는 방법밖에 없는데 시간복잡도가 O(N)이다.
    • 딕셔너리를 사용하거나 set()을 사용하면 시간복잡도가 O(1)
