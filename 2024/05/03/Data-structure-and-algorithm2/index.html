자료구조와 알고리즘2

May 03, 2024 in Algorithm

스택, 큐, 트리, 힙에 대해 알아보자
프로그래머스 강의인 “어서와! 자료구조와 알고리즘은 처음이지?” 강의를 보고 정리한 내용입니다.

스택

  • 자료를 보관할 수 있는 선형 구조
  • 후입선출(LIFO, Last-In First-Out)의 특징을 가짐
  • 스택 언더플로우: 비어 있는 스택에서 데이터 원소를 꺼내려할때
  • 스택 오버플로우: 꽉 찬 스택에 데이터 원소를 넣으려 할때

스택의 추상적 자료구조 구현

  1. 배열을 이용하여 구현
    • 파이썬의 리스트와 메서드들 이용
  2. 연결리스트를 이용하여 구현
    • 양방향 연결 리스트 이용

큐

  • 자료를 보관할 수 있는 선형 구조
  • 선입선출(FIFO, First-In First-Out)의 특징을 가짐
  • 인큐, 디큐

큐의 추상적 자료구조 구현

  1. 배열을 이용하여 구현
    • 파이썬의 리스트와 메서드들 이용
  2. 연결리스트를 이용하여 구현
    • 양방향 연결 리스트 이용

우선순위 큐

  • FIFO 방식을 따르지 않고, 원소들의 우선순위에 따라 큐에서 빠져나오는 방식
  • 예를 들어, 운영체제의 CPU 스케줄러

트리

  • node, edge를 이용하여 데이터의 배치 형태를 추상화한 자료구조
  • 노드 (nodes)
  • 간선 (edges)
  • 루트 노드 (root node), 리프 노드 (leaf nodes), 내부 노드 (internal nodes)
  • 부모 (parent) 노드와 자식 (child) 노드
  • 노드의 수준 (level)
  • 노드의 차수 (degree)
  • 트리의 높이 (height) - 또는, 깊이 (depth) -
  • 부분 트리 (서브트리; subtrees)
  • 이진 트리 (binary trees)
  • 포화 이진 트리 (full binary trees): 모든 레벨에서 노드들이 모두 채워져 있는 이진 트리, 높이가 k이고, 2의 k승 -1인 이진 트리
  • 완전 이진 트리 (complete binary trees): 높이 k인 완전 이진트리는 레벨 k-2까지는 모든 노드가 2개의 자식을 가진 포화 이진 트리이고, 레벨 k-1에서는 왼쪽부터 노드가 순차적으로 채워져 있는 이진트리임

트리 종류

  • 이진 트리: 모든 노드의 차수가 2 이하인 트리

BFS(깊이 우선 순회, depth first traversal)

  • 중위 순회 in-order traversal: left subtree > 자기 자신 > right subtree
  • 전위 순회 pre-order traversal: 자기 자신 > left subtree > right subtree
  • 후위 순회 post-order traversal: left subtree > 자기 자신 > right subtree

BFS(넓이 우선 순회, breadth first traversal)

  • 규칙
    • 수준이 낮은 노드를 우선으로 방문
    • 같은 수준의 노드들 사이에는, 부모 노드의 방문 순서에 따라 방문, 외쪽 자식 노드를 오른쪽 자식보다 먼저 방문
  • 한 노드를 방문했을 때, 나중에 방문할 노드들을 순서대로 기록해두어야 함 - 큐를 이용

이진 탐색 트리

  • 모든 노드에 대해서, 왼쪽 서브트리에 있는 데이터는 모두 현재 노드의 값보다 작고, 오른쪽 서브트리에 있는 데이터는 모두 현재 노드의 값보다 큰 성질을 만족
  • 배열을 이용한 이진 탐색과 비슷
  • O(log n)의 시간 복잡도를 가지는데 항상 그러한것은 아니다. 한쪽으로 치우쳐진 트리의 경우 선형탐색과 비슷한 시간복잡도를 가지게됨
  • 연산: insert, remove, lookup, inorder(중위순회), min, max

힙

  • 이진 트리의 한 종류
  • 원칙
    • 루트 노드가 언제나 최댓값 또는 최솟값을 가짐
      • 최대 힙, 최소 힙
    • 완전 이진 트리여야 함
  • 종류
    • 최대 힙
    • 최소 힙

이진 탐색 트리 VS. 최대 힙, 최소 힙

  • Question 1: 원소들은 완전히 크기 순으로 정렬되어 있는가?
    • 이진 탐색 트리는 yes
    • 최대 힙, 최소 힙은 no, 느슨하게 정렬되어 있음
  • Question 2: 특정 키 값을 가지는 원소를 빠르게 검색할 수 있는가?
    • 이진 탐색 트리는 yes
    • 최대 힙, 최소 힙은 no
  • Question 3: 부가의 제약 조건은 어떤 것인가?
    • 힙은 이진 탐색 트리에 비해 완전 이진 트리어야한다.
