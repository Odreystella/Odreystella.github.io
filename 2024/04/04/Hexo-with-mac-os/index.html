hexo) mac환경에서 hexo 이어서 블로깅하기(hexo 배포 원리 이해하기)

Apr 04, 2024 in Study, Hexo

mac 환경에서 hexo 블로그 이어서 사용하기
결론: 백업해둔 파일이 없으면 새로 시작해야한다.

  1. 준비환경 만들기
    1. homebrew, git가 설치되어 있는지 확인하기
    2. nvm 설치하기
      • Node Version Manager
      • 하나의 OS에서 여러 버전의 node.js를 설치하고, 사용할 버전을 쉽게 전환할 수 있게 도와주는 프로그램이다. pyenv와 같은 역할이다.
    3. Node.js 설치하기
    4. Hexo 설치하기
  2. nvm 설치하기
    • brew install nvm
    • nvm -v 했을 때 command not found:nvm이 뜬다.
    • zshrc에 환경변수 추가하기
      1. vi ~/.zshrc
      2. 아래 내용을 추가하고 변경사항을 적용한다.
      3. 이젠 버전확인이 가능할 것이다. 버전 확인해보니 10.5.0이 설치되었다.
  3. Node.js 설치하기
    • nvm install --lts v20.12.1이 설치되었다.
    • nvm install v18.15.0 으로 특정 버전을 설치할 수도 있다.
  4. Hexo 설치하기
    • npm install -g hexo-cli -g는 global
    • Hexo는 정적 사이트 생성기(Static Site Generator)이다.
  5. 환경 셋팅이 끝났으니 hexo init 을 하고 기존에 블로그 올려둔 내용을 깃허브에서 클론하면 되겠다고 생각했는데, 클론한 폴더 구조가 내가 생각했던 거랑 달라서 검색이 필요했다. 깃허브에 올라가있는 파일들은 *.md 파일들이 아니라 public 폴더에 있는 파일들이었다.
  6. 이쯤에서 hexo의 배포과정에 대한 이해가 필요했다.
    1. hexo new 커맨드로 새 글을 작성한다. 이때 *.md 파일이 생성된다.
    2. hexo generate 또는 hexo g 커맨드로 빌드한다. 이때 *.md 파일을 웹사이트(html, css, js)로 만들어준다. 이 결과물이 public 폴더에 저장된다.
    3. hexo deploy 또는 hexo d 커맨드로 배포한다. github.io 도메인으로 새로 올린 글을 확인할 수 있다.
    4. 결과적으로 내가 작성한 *.md 파일들은 깃허브에 올라가지 않는다. 수정한 템플릿과 설정파일, 테마도 따로 저장되지 않는다. 이런 부분을 위해서는 어딘가에 백업이 필요하다.
  7. 정말 다행히 나는 예전에 의도치않게 백업해둔 폴더가 있어서 그걸 가져와서 hexo server 를 돌려보니 기존처럼 잘 실행됐다. 백업된 폴더가 없었으면 다시 만들었어야 했을텐데 참 다행이다..열심히 블로깅해야겠다.
