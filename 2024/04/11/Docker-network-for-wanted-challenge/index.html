Docker Command

Apr 11, 2024 in Docker

원티드 4월 백엔드 챌린지 2차 세션을 듣고 자주 사용하는 도커 명령어에 대해 정리해보았다.

Docker System 명령어

  1. 버전 및 상세정보 조회하기
    • docker version
  2. 버전 및 빌드 버전 한줄로 조회하기
    • docker -v
  3. 시스템 전체 정보 확인하기
    • docker system info
  4. 디스크 사용량 확인하기
    • docker system df
  5. 도커 허브에 로그인하기
    • docker login -u {유저 ID}

Docker Image 명령어

  1. 이미지 목록 확인하기
    • docker images 또는 docker image ls
  2. 이미지 빌드하기
    • Dockerfile를 이용해서 이미지 만들기
    • docker build -t {이미지명}:{태그명} {도커파일 경로}
  3. 이미지 상세 정보 확인하기
    • docker image inspect {이미지명}
    • 도커 이미지가 어떠한 정보로 만들어졌는지 역 추적해 볼 수 있다.
    • Created 생성 날짜, ExposedPorts, Env를 통한 Apache 구성 정보, CMD, ENTRYPOINT와 같은 기동 시점의 실행 정보들, WorkDir, Volume 정보 등을 모두 확인 할 수 있다.
  4. 도커 허브에서 이미지 검색하기
    • docker search {mysql}
  5. 도커 허브에서 이미지 가져오기
    • docker pull {이미지명:태그}
    • docker pull ubuntu:20.04
  6. 이미지 삭제하기
    • docker rmi {이미지 ID}

Docker Container 명령어

  1. 현재 실행중인 컨테이너 확인하기
    • docker ps
  2. 모든 컨테이너 확인하기
    • 정지된 컨테이너도 확인할 수 있다.
    • docker ps -a
  3. 컨테이너 쉘 접속하기
    • docker exec -it {컨테이너 ID} /bin/bash
  4. 컨테이너 삭제하기
    • 컨테이너가 중지된 상황에만 삭제할 수 있다.
    • docker rm {컨테이너 ID}
  5. 컨테이너 시작/중지/재시작하기
    • docker start {이미지 ID}
    • docker stop {이미지 ID}
    • docker restart {이미지 ID}
  6. 컨테이너 실행하기
    • docker run --name mydocker -it ubuntu:18.04
    • docker run -d -p 8080:8080 -t spring-helloworld
  7. 사용중인 컨테이너를 이미지로 만들기
    • docker commit {컨테이너명} {이미지명}
  8. 호스트에서 컨테이너로 파일 복사하기
    • docker cp {호스트 파일 경로} {컨테이너 이름}:{컨테이너 파일 경로}
  9. 컨테이너에서 호스트로 파일 복사하기
    • docker cp {컨테이너 이름}:{컨테이너 파일 경로} {호스트 파일 경로}
