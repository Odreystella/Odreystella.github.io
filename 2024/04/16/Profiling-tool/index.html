프로파일링과 silk

Apr 16, 2024 in DRF

프로파일링과 silk 사용법에 대해 알아보자

프로파일링(Profiling)

  • 코드 자체만으로 성능을 예측할 수 없기 때문에 프로그램을 실행하면서 성능과 리소스 사용을 측정하고 분석하는 방법이다.
  • 소스 레벨에서 실행 속도를 측정하고 개선하기 위한 기술로, 어떤 함수가 얼마나 시간이 걸리는지, 어디서 병목이 발생하는지, CPU/메모리를 얼마나 사용하는지 등의 성능을 측정하여 최적화할 방법을 찾는다.
  • 성능을 분석하는 툴로 개발 환경에서는 Profiling Tool, 운영 환경에서는 APM Tool 을 사용한다.

APM Tool

  • Application Performance Monitoring/Management
  • 안정적인 웹 애플리케이션 운영을 위해 성능 및 장애를 관리한다.
  • 프론트엔드, 백엔드, 인프라, 네트워크 등 다양하게 모니터링 할 수 있다.
  • Top 10 Open Source APM Tools

퍼포먼스 벤치마킹

  • 주로 경쟁사 분석과 성능 비교를 목적으로 한다.
  • 여러 제품 또는 서비스 간의 성능을 비교하기 위해 특정 작업이나 시스템의 전반적인 성능을 측정한다.
  • 여러 가지 표준화된 벤치마킹 도구나 테스트 슈트를 사용하여 성능을 측정하고 결과를 비교한다.

성능 테스트 종류

  1. Load Testing
    • 목표로 설정한 부하 상황에서 시스템 성능을 검증한다.
    • 일반적으로 1시간동안 유지되며, 최대 부하 트래픽을 감당할 수 있는지 확인한다.
  2. Stress Testing
    • 일반적인 부하 테스트보다 훨씬 높은 부하 상황에서 시스템 성능을 검증한다.
    • 일반적으로 부하테스트의 2배를 사용하며, 시스템 장애가 발생하는지, 정상 수준으로 돌아갈 때 복구되는지 확인한다.
  3. Soak Testing
    • Endurance Testing으로도 불리며 8시간 이상의 기간에 걸쳐 시스템의 안정성을 확인하는 장기적인 테스트이다.
    • 메모리 누수와 같은 이슈들을 감지한다.
  4. Spike Testing
    • 빅세일, 신제품 출시 등 사용량 급증이 발생할 이벤트 상황에서 시스템의 안정성을 확인한다.
  5. Breakpoint Testing
    • 동시 접속자 수를 증가시켜 시스템의 장애 지점을 결정하는 테스트이다.

웹 서비스에서 성능 측정 항목

  • TPS(Transaction Per Second): 초당 트랜잭션 개수
  • TTI(Time To Interactive): 페이지가 완전히 상호작용이 가능하게 되기까지 소요된 시간, 3.8초 이내가 권장된다.
  • TTFB(Time To First Byte): 리소스 요청의 응답에 대해 첫번째 바이트를 수신하기까지 소요된 시간, 0.6~0.8초 이내가 권장된다.
  • LCP(Largest Contentful Paint): 페이지가 로드되기 시작한 시점부터, 가장 큰 콘텐츠(이미지, 비디오 등)가 렌더링될 때까지의 시간, 2.5초 이내가 권장된다.
  • FCP(First Contentful Paint): 페이지가 로드되기 시작한 시점부터, 화면에 처음으로 콘텐츠가 렌더링될 때까지의 시간, 1.8초 이내가 권장된다.
  • Throughput(처리량): 시간당 서버가 처리하는 요청의 양
  • Concurrent User(동시 사용자 수): 동시에 서버에 접속하여 요청을 보내는 사용자의 수
