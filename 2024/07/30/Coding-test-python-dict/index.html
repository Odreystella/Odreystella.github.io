코딩테스트 합격자되기 파이썬편 - 딕셔너리

딕셔너리의 특징

  • 해시 기반으로 이루어진 자료구조
  • {}를 사용해서 선언한다.
  • {키1: 값1, 키2: 값2}처럼 키-값 쌍으로 되어 있다.

해시란?

  • 해시 테이블
  • 해시 함수값이 같아서 충돌나면?
  • 해시 함수를 활용해서 키값을 인덱스로 만든다.

딕셔너리 생성과 키, 값 추가

  • {}나 dict 함수를 이용하여 생성

딕셔너리 접근

  • 리스트의 경우 어떤 원소를 찾을 때 순차탐색을 하기 때문에 시간 복잡도가 O(N)인데, 딕셔너리는 O(1)을 가짐
  • dict.get(key, default): 해당하는 키가 없는 경우 None 반환
  • dict[key]: 해당되는 키가 없는 경우, KeyError 발생. 예외처리 해줘야 함

딕셔너리 값 변경

  • dic = {‘a’: 1, ‘b’: 2, ‘c’: 3}
  • dict['c'] = 1

딕셔너리 메서드와 시간복잡도

  • dict.upadte(dic): 딕셔너리를 추가할 수 있음
    • 추가할 dic의 크기가 K라면, 시간복잡도 O(K)
  • dict.keys(): 모든 키값을 가져옴
    • 메서드 자체 시간복잡도는 O(1)
    • 딕셔너리 크기가 N일 때, 반복문을 돌며 모든 키값을 가져올 때는 시간복잡도 O(N)
  • dict.items(): 모든 키, 값을 가져옴
    • 메서드 자체 시간복잡도는 O(1)
    • 딕셔너리 크기가 N일 때, 반복문을 돌며 모든 키, 값을 가져올 때는 시간복잡도 O(N)
  • del dic['key']: 키에 해당하는 버킷 삭제
    • 시간복잡도 O(1)
    • 반환값 없음
  • dict.pop(key): 키에 해당하는 버킷 삭제
    • 시간복잡도 O(1)
    • 삭제된 키에 해당하는 값 반환함

코딩테스트에서 딕셔너리를 사용할 때 주의할 점

  • 뮤터블 객체(list, set, dictionary)는 key가 될 수 없다.
    • KeyError 발생
  • dict['key'] 방식으로 딕셔너리에 접근할 때, key가 존재하지 않으면 KeyError이 발생한다.
    • dict.get(key, default) 방식을 사용하자. KeyError가 안나게 디폴트값을 반환하는 것이고, 존재하지 않는 key를 추가하는 것은 아니다.
    • dict.setdefault(key, default)는 key가 존재하지 않을 때, 딕셔너리에 새로운 key와 default를 추가하는 메서드이다.
  • 중복 key를 허용하지 않는다.
    • 키에 해싱함수를 적용한 값을 인덱스로 활용하기 때문에 같은 값이 key로 존재한다면 구분할 수가 없음.
    • 중복 key가 있다면 list, deque, tuple 같은 자료구조를 고려
